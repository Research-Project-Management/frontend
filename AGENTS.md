# Agent Instructions for Flux

## Development Commands

```bash
pnpm dev              # Start dev server on port 2916
pnpm build            # Production build
pnpm start            # Run production build on port 2916
pnpm typecheck        # Run: react-router typegen && tsc
```

**IMPORTANT**: No linting, formatting, or test commands are configured in this repo. Do not assume they exist.

## Architecture

- **Framework**: React Router 7 SPA (SSR disabled in `react-router.config.ts`)
- **Package Manager**: pnpm
- **Port**: 2916 (both dev and production)
- **Backend API**: Configured via `VITE_API_URL` env var (default: `http://localhost:2915`)

## Key Directories

- `app/routes/` - Route files (file-based routing, React Router 7 convention)
- `app/components/` - React components organized by feature (auth, workspace, shared, ui)
- `app/query/` - TanStack Query hooks for server state
- `app/hooks/` - Custom React hooks (auth, upload, workspace, socket)
- `app/lib/` - Utilities (`api.ts` - centralized fetch, `socket.ts` - socket.io client)
- `app/stores/` - Zustand stores for client state
- `app/contexts/` - React contexts (SocketProvider, ChatModeContext)
- `app/types/` - TypeScript type definitions

## Important Conventions

- **Type imports**: Use `import type { Route } from "./+types/route-file-name"` for route types (generated by `react-router typegen`)
- **Path aliases**: `~/*` and `@/*` both map to `./app/*`
- **API calls**: All API requests go through `app/lib/api.ts` (`apiGet`, `apiPost`, `apiPut`, `apiDelete`)
- **Query client**: Singleton in `app/lib/react-query.tsx`, wrapped in `QueryProvider` at root
- **Socket.io**: Singleton in `app/lib/socket.ts`, wrapped in `SocketProvider` at root
- **shadcn/ui**: Components in `app/components/ui/`, configured via `components.json`
- **Route meta**: Export `meta()` function for SEO/meta tags in route files

## Build Artifacts

- Output: `build/` directory (contains `client/` static assets, no `server/` since SSR is disabled)
- Generated types: `.react-router/types/` (run `pnpm typecheck` to regenerate)
- Generated types are included in tsconfig via `.react-router/types/**/*`

## Deployment

- CI workflow: `.github/workflows/main.yml` (only SSH deploy step active)
- Production: Builds and deploys via SSH to `/root/app/Flux`, restarted with PM2
- Dockerfile uses multi-stage build with pnpm, runs on Node 20 Alpine

## Dependencies

- **State**: TanStack Query v5 (server), Zustand (client)
- **Real-time**: socket.io-client
- **UI**: Radix UI primitives, Lucide icons, shadcn/ui, TailwindCSS 4
- **Editors**: TipTap (rich text), Monaco Editor (code)
- **Auth**: @react-oauth/google
- **DnD**: @dnd-kit

## Missing/Not Configured

- No ESLint configuration
- No Prettier configuration
- No test framework (no Jest, Vitest, or test scripts)
- No pre-commit hooks
